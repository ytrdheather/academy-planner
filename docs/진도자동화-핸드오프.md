## 1. 핵심 개념 (숙제 세팅 로직)

### 1-1. 데이터 모델: "과목별 · 등원요일별 개수 맵"
- 등원일에 내주는 숙제 = **"다음 등원까지 완료해야 할 최소단위(지문) 개수"**.
- 이 개수는 **마감일 기준**으로 표현: `월:2` = "월요일 수업 올 때까지 2개 완료".
- 숙제 출력 예: **"월요일(7/20)까지 2개 해오기"** (요일 + 날짜 병기).

### 1-2. 단위(1개 = 지문 1행)
- "1개" = 교재 세부 내용 DB의 **`제목` 1행**(가장 작은 학습단위 = 1지문/1lesson).
- 워크북은 별도 커서 없이 **주교재 유닛을 그대로 따라감**(없으면 생략).

### 1-3. 입력 3모드 (엔진이 요일별 맵을 자동 생성)
| 모드 | 의미 | 계산 |
|---|---|---|
| **등원마다 N** | 나올 때마다 N개 | 각 등원요일 = N |
| **매일 N** | 하루 N개 페이스 | 직전 등원 다음날 ~ 이번 등원일, **일요일만 제외**, 하루 N개 (주간합 6N) |
| **불규칙** | 요일별 직접 지정 | 텍스트 "월2 수1 금1" |

**매일 모드 예시 (하루 1개):**
- 월수금 → 월2·수2·금2 (합6)
- 화목 → 화4·목2 (합6)
- 월금 → 월2·금4 (합6)
- 월수 → 월4·수2 (합6)

### 1-4. 커서 = 지문 순번 (1-based)
- `{과목}현재유닛` 칸 = **다음에 풀 지문의 순번**(교재 세부 내용 DB의 `순번`).
- 유닛당 1지문 교재(대부분) → 순번 = 유닛번호. "Unit 6부터" → 커서 `6`.
- 유닛당 2지문 교재 → Unit 6 첫 지문 = 순번 `(6-1)×2+1 = 11`.
- 비워두면 엔진이 `1`(처음)부터 시작.

### 1-5. 확정된 정책 결정 (2026-07-15)
- **기록 위치**: 오늘(배정일) 행. (학생 플래너가 today 행을 읽으므로 지금 바로 보임)
- **커서 의미**: 지문 순번(전체 순서).
- **중복 방지**: 확정은 **빈 숙제칸에만** 기록(이미 있으면 스킵) → 두 번 눌러도 커서 이중 전진 없음.
- **주/부/어휘 완전 독립**, 부교재 없으면(relation 빈칸) 그 과목 스킵.

### 3-7. 숙제 미룸/이월(↩) — 2026-07-15 추가
- **문제**: 확정 시 커서가 전진하므로, 학생이 특정 과목 숙제를 안 하면 그 범위가 지나가버림.
- **해법(A방식: 밀린 것만)**: 그 과목 커서를 **직전 배정량만큼 되돌림** → 다음 생성 때 같은 범위가 새 마감으로 재출제. 따라잡을 때까지 새 진도는 멈춤.

- **백엔드**: preview 과목 출력에 `advanced`·`lastAmtField` 추가. `POST /api/defer-homework {name, prefix}`(prefix∈어휘/주독해/부독해): 명부 이름조회 → 커서 −= 직전배정량(최소1), 직전배정량 0으로(중복 미룸 방지).
- **프론트**: 출결·숙제 탭 어휘/주독해/부독해 숙제칸마다 `↩` 버튼 → `deferHomework(name, prefix)`. (문법칸은 없음)
- **라이브 검증**: 홍상연 설정→확정(현재유닛3·직전배정량2)→미룸(커서3→1·직전배정량0)→중복미룸 거부→원복.

## 4. 남은 작업 (설계 확정, 미구현)

### 4-1. ③ 학습상태 연동
`학습상태 ≠ 정상`일 때 3가지 자동 동작:
1. **숙제 생성 정지** — ✅ 이미 엔진에 반영됨(빈값=정상 취급).
2. **누락 점검 제외** — ❌ 현재는 결석사유만 제외. 학습상태 기준도 추가 필요(`computeMissing`·`buildStatusBadges`).
3. **사유 배지** — ❌ 이름 옆에 `🏖️ 여행휴가`·`💤 휴원` 등 배지(현재 `🚫 결석` 배지처럼).

### 4-2. 커서 힌트 (중간 시작 안전장치)
- 진도 관리 탭 커서 칸 옆에 **"다음에 풀 지문: Unit 6 - Reading 1"** 실시간 표시.
- 커서가 순번이라, 선생님이 숫자 입력 시 어느 지문인지 즉시 확인 → 중간 합류 학생 세팅 실수 방지.

### 4-3. 휴일 연동 (확정 결정 반영)
- **결정**: (a) 하이브리드 — 공휴일 자동 프리필 + 학원 휴무일 DB 수동 조정. (b) **휴일도 숙제일로 셈**(개수 공식은 일요일만 제외 유지).
- **결과적으로 휴일은 "마감일"만 건드림**:
  - `nextClassInfo` 규칙 변경: "다음 등원일 = 수강요일 중 **휴무일이 아닌** 첫 개원일".
  - 월 수업이 휴일이면 마감이 "수요일까지"로 이동. 매일 모드는 실제 날짜 기반 계산이라 그 사이 날수만큼 개수 자동 증가. 등원마다는 날짜만 이동.
- **구현 요소**:
  1. 노션 `학원 휴무일` DB(날짜 목록) — 진실의 원천.
  2. 공휴일 프리필 — 오프라인 npm `date-holidays`(음력·대체공휴일 내장) 추천(공공데이터포털 API 키 발급 불필요). 원장이 조정하는 초안.
  3. 엔진 보강: `nextClassInfo`가 휴무일 셋을 받아 건너뜀 + 매일 개수를 **요일 패턴 기반 → 실제 날짜 기반**으로 전환.
- 진도 관리 탭의 요일별 프리뷰는 "평상시 한 주" 예시로 그대로 유지(역할 분리).

## 5. 알려진 이슈 / 주의사항

- ⚠️ **교재 relation 정리 필요**: 다수 학생의 `어휘교재` relation에 여러 책이 섞여 있음(레거시). 엔진은 `relation[0]`만 사용. **실사용 전 과목별 단일 교재로 정리** 필요.
- 오늘(생성 시점) 일일행이 있어야 미리보기 대상이 잡힘(행은 아침 10시 자동 생성).
- Windows 셸에서 curl `-d`에 한글/이모지 속성명을 직접 넣으면 인코딩 깨짐 → payload는 파일로 만들어 `--data-binary @file` 사용.

## 6. 이어서 작업하는 법

### 6-1. 환경
- 저장소: `github.com/ytrdheather/academy-planner`, 기본 브랜치 `main` (Render 자동 배포).
- 시작 스크립트: `node api/index.js` (포트 `PORT` env, 기본 5001).
- 배포용 실 코드는 **루트** `api/index.js` + `public/views/teacher.html`. (`academy-planner/uri-hagweon-gwanri-peurogeuraem/` 하위는 별개 — 건드리지 말 것.)

### 6-2. 로컬 테스트 팁
- 서버: `node api/index.js` → `http://localhost:5001/teacher`.
- 교사 페이지는 `localStorage['teacher_auth_token']` 필요(JWT). 로컬 검증 시 `.env`의 `JWT_SECRET`으로 매니저 토큰 발급해 주입.
- 라이브 노션 데이터로 검증할 땐 **반드시 자기복구(원복)** 방식으로(위 3-5 참고).
